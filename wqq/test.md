## Задание 1  
**Ситуация:** Проблемы сетевого подключения на рабочей станции.  
**Задача:** Пользователь жалуется на отсутствие сетевого подключения на своей рабочей станции.  
**Решение:**  
1. Проверка связи с шлюзом:  
```

ping \<IP\_шлюза>

```
2. Проверка DNS-сервера:  
```

ping \<IP\_DNS\_сервера>

```
3. Проверка доступа к внешнему ресурсу (например, ya.ru):  
```

ping ya.ru

```
4. Если пинги не проходят — проверяем физическое подключение, настройки IP, перезагружаем сетевой адаптер, проверяем настройки DHCP.

## Практические задания

### Вариант 1

**Задача:** Настроить статический IP-адрес на Windows и проверить связность.
**Шаги:**

1. Открыть «Сетевые подключения».
2. Выбрать адаптер, открыть свойства TCP/IPv4.
3. Ввести IP, маску, шлюз и DNS.
4. Проверить связь с соседними устройствами командой `ping`.

### Вариант 2

**Задача:** Создать два VLAN на коммутаторе и изолировать трафик.
**Шаги:**

1. Создать VLAN 10 и VLAN 20:

   ```
   vlan 10
   vlan 20
   ```
2. Назначить порты к VLAN:

   ```
   interface FastEthernet0/1
   switchport mode access
   switchport access vlan 10
   interface FastEthernet0/2
   switchport mode access
   switchport access vlan 20
   ```
3. Проверить изоляцию (ping между VLAN).
